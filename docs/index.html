Quick Start

# Clone the repository
git clone https://github.com/kamilrybacki/ansible.git && cd ansible

# Install dependencies
pip install -r requirements-test.txt
ansible-galaxy collection install -r requirements.yml

# Run any playbook
ansible-playbook ai/dify-setup/setup.yml

# Optional: Set up Vault first for centralized secrets
ansible-playbook security/vault-setup/setup.yml

Single Sign-On (SSO)

Authenticate once — access everything. Authelia serves as the central identity provider for the entire homelab.

How it works

  • WireGuard / LAN: Authelia requires only one factor (username + password). No TOTP prompt.
  • WAN: Full two-factor (password + TOTP) as normal.
  • Session duration: 30 days — log in once, stay logged in.
  • OIDC services: After logging in to Authelia once, clicking any OIDC-connected service auto-creates your session with no additional credential prompt.
  • Header-auth services: Caddy injects the Remote-User header after Authelia validates you; the service auto-logs you in.

Service SSO method

Service Method Notes
GrafanaOIDCAuto-login; GrafanaAdmin role for admin group
OutlineOIDCAuto-wired to Authelia on deployment
VaultwardenOIDC (SSO)Requires Vaultwarden v1.30+
Paperless-ngxHeader authPAPERLESS_ENABLE_HTTP_REMOTE_USER
NetBoxHeader authREMOTE_AUTH_ENABLED
LibreNMSHeader authAUTH_MECHANISM=header
Uptime Kuma, n8n, Dify, Stirling-PDFAuthelia portalOne login to Authelia, no native SSO

OIDC endpoints

  • Authorization: https://auth.<domain>/api/oidc/authorization
  • Token: https://auth.<domain>/api/oidc/token
  • Userinfo: https://auth.<domain>/api/oidc/userinfo
  • Discovery: https://auth.<domain>/.well-known/openid-configuration

Adding a new service

To add another OIDC client, add it to security/secure-homelab-access/roles/authelia/templates/configuration.yml.j2 under identity_providers.oidc.clients and re-run the secure-homelab-access playbook. The client secret is auto-generated and saved to ~/.homelab-setup-vars.yml.

Vault Integration
